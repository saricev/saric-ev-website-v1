## 二、架构一致性问题

### CRITICAL — 双数据源冲突

前端 8 个组件直接 import `@/data/` 静态文件，后台管理用 SQLite。**管理员修改数据后前端不会更新**。

受影响组件：

| 组件 | 导入来源 |
|------|----------|
| `src/components/layout/Header.tsx:6` | `@/data/company` |
| `src/components/layout/Footer.tsx:2` | `@/data/company` |
| `src/components/home/StatsBar.tsx:2` | `@/data/company` |
| `src/components/ui/WhatsAppButton.tsx:1` | `@/data/company` |
| `src/components/forms/ContactForm.tsx:8` | `@/data/products` |
| `src/app/contact/page.tsx:7` | `@/data/company` |
| `src/app/about/page.tsx:8` | `@/data/company` |
| `src/components/products/ProductFilter.tsx:3` | `@/data/products` |

**修复方向**: 改为 Server Components 从 `@/lib/data` 读取 SQLite 数据。

## 四、验证结果

| # | 发现 | 验证结论 |
|---|------|----------|
| 1 | SQL 注入 | **确认** — 表名有白名单，列名无过滤 |
| 2 | GET 无鉴权 | **确认** — 零 auth 检查 |
| 3 | JWT 密钥硬编码 | **确认** — 生产有 guard，其他环境静默使用弱密钥 |
| 4 | 默认密码 admin123 | **确认** — 两处回退 + 控制台打印 |
| 5 | 双数据源冲突 | **确认** — 前端读静态文件，admin 改 SQLite |
| 6 | 表单无校验 | **确认** — 只检查字段存在性 |
| 7 | 上传 DELETE 无鉴权 | **修正** — 有 auth 检查，缺的是细粒度权限 |
